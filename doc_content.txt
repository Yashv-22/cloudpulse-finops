PART I — PRODUCT REQUIREMENTS DOCUMENT  (PRD V2.0)
Product Requirements Document
Autonomous AI FinOps & Cloud Governance Platform — AWS Edition
1.  Executive Summary
CloudPulse Pro
An autonomous B2B SaaS platform designed to bridge the gap between cloud financial management (FinOps) and security governance. By leveraging AI-driven agents, the platform identifies cost-leakage, security vulnerabilities, and carbon inefficiencies — providing Zero-Touch autonomous remediation via Infrastructure-as-Code (Terraform).

3.  Functional Requirements
3.1  FinOps & Cost Optimization
FR-1
Zombie Resource Hunter: Automated scanning for unattached EBS volumes, idle ALBs, and orphaned RDS snapshots using Boto3 queries against AWS EC2, ELB, and RDS APIs.
FR-2
Cost Explorer Integration: Pull daily and monthly granular spend data via the AWS Cost Explorer API (ce). Visualize spend trends with Recharts/Tremor on the dashboard.
FR-3
Savings Recommendations: Logic engine to suggest switching idle On-Demand instances to Spot or Reserved Instances, with projected monthly savings estimates.
3.2  Autonomous Governance
FR-4
Live Compliance Index: A real-time score (0-100%) calculated from security posture indicators: S3 bucket encryption status, MFA enforcement, public access blocks, and more.
FR-5
Security Group Auditing: Detection and flagging of overly permissive rules — specifically Port 22 (SSH) and Port 3389 (RDP) open to 0.0.0.0/0.
FR-6
AI Remediation Engine: LangChain-powered agent that generates production-grade Terraform scripts to fix detected security risks. One-click staged deployment with human approval gate.
3.3  GreenOps (Sustainability)
FR-7
Carbon Tracker: Estimate the carbon footprint of AWS resources based on region, instance type, and utilization metrics sourced from AWS CloudWatch.
FR-8
Regional Arbitrage: Suggest migration of non-critical workloads to Green Regions (lower carbon intensity per kWh) with automated migration cost-benefit analysis.

9.  System Architecture Design
9.1  The Agent + Event Loop Pattern (2026 Standard)
CloudPulse Pro uses an Async Event Fabric rather than traditional request-response. This enables real-time streaming of AI reasoning to the dashboard.
Step
Stage
Description
1
Scanner Trigger
Cron job or user action triggers a Lambda/Celery background task.
2
AWS Audit
Boto3 queries AWS Config and Cost Explorer APIs for resource metadata.
3
AI Planning
LangChain analyzes results. If a Zombie is found, it creates a Remediation Plan event.
4
UI Streaming
Backend uses Server-Sent Events (SSE) to stream the AI thought process to the glass-card terminal in real-time.
10.  Logical Data Flow — Zombie Resource Detection
Step 1: Identification Queries
EBS
Query all volumes with State: available (unattached). Flag volumes with last-attach-time > 30 days.
RDS
Identify automated snapshots older than X days with no associated active DB instance ARN.
ELB / ALB
Flag Application Load Balancers where CloudWatch RequestCount = 0 over the trailing 7-day window.
Step 2: AI Remediation Prompt Template
LangChain System Prompt
"You are a CloudPulse Pro AI Agent. Analyze this AWS resource: {resource_metadata}. Generate a Terraform destroy or modify script to optimize cost while maintaining a 0% risk profile. Include a Reasoning block explaining the decision, an Estimated Monthly Savings figure, and a Confidence Score (0-100)."
Step 3: Verification Gate
All AI-generated Terraform code is passed through a terraform validate check in an isolated sandbox container before being surfaced to the user. Output is never shown if validation fails.
11.  Security & Permission Design
Principle of Least Privilege
Since the application has the ability to modify and delete production infrastructure, the permission model follows strict least-privilege design with two distinct IAM modes.
Mode
IAM Policy
Activation
Scope
Audit Mode
arn:aws:iam::aws:policy/ReadOnlyAccess
Always active during scanning and dashboard refresh.
List and Describe permissions only. No writes.
Remediation Mode
Custom Scoped Policy
Only activated when user explicitly clicks Execute Fix.
Targets specific resource IDs via temporary STS token. Expires after 1 hour.

13.  Frontend Layer
Next.js 14+ (App Router)
React-based framework powering the dashboard, resource explorer, recommendations, and compliance views.
Why Next.js 14
SSR ensures fast initial loads for data-heavy dashboards. App Router provides robust structure for SSE-driven AI streaming views.
Key Pages
Landing / Login / Signup / Dashboard / Resources / Recommendations / Cost Explorer / Settings
Rendering Strategy
Server Components for read-heavy pages (resources, cost). Client Components for interactive elements (gauge, approval modals).
Glassmorphism Implementation
Blur Effect
backdrop-filter: blur(16px) on all card surfaces. Requires webkit prefix for Safari compatibility.
Transparency
background: rgba(255,255,255,0.05) with border: 1px solid rgba(255,255,255,0.10).
Glow Effects
box-shadow: 0 0 20px rgba(59,130,246,0.3) for blue glow; 0 0 20px rgba(168,85,247,0.3) for purple.
Dark Base
#0a0a0c background with radial-gradient mesh (blue-to-purple) at z-index 0.
14.  Backend Layer
FastAPI (Python 3.11+)
High-performance ASGI framework serving as the intelligence hub and API gateway.
Why FastAPI
Native async support handles long-running Boto3 scans and LLM calls without blocking the event loop.
Key Routers
/auth (JWT) · /dashboard (KPI aggregation) · /resources (scan + filter) · /remediation (Terraform gen + execute)
Background Tasks
Celery workers consume tasks from Redis queues for scans that may run 30-120 seconds.
Retry Policy
Exponential backoff, max 3 retries for AWS API failures. Dead-letter queue for persistent errors.

15.  AI & Automation Integration
LangChain Orchestration
Why LangChain
Abstracts LLM provider switching (OpenAI GPT-4o <-> Claude 3.5 Sonnet via Bedrock) and provides reusable chain primitives.
Key Chains
ResourceAnalysisChain · TerraformGenerationChain · SavingsReasoningChain · ComplianceExplainerChain
AI Logic Loop
1. Extract JSON metadata of Zombie Resource. 2. Send to LLM via LangChain with structured system prompt. 3. Validate generated Terraform in sandbox. 4. Surface to user approval queue.
Cost Optimization
Llama 3 (Ollama, local) handles lightweight metadata classification tasks to avoid LLM API costs for simple heuristics.

Implementation Note — Starting Point
Recommended First Step for Security Engineers
Begin with the Backend IAM Policies before writing any application code. Define a 'Security Auditor' IAM Role with List and Describe permissions only, and explicitly Deny all Delete and Modify actions. This ensures the AI remediation engine cannot accidentally wipe production infrastructure during development and testing phases. Only add scoped Write permissions once the heuristic scoring engine has been validated against a staging environment.
